Custom codec used in layer <-> proxy communication. Supports both synchronous (required by the layer) and asynchronous (convenient for the proxy) IO. Asynchronous IO is feature-gated with the codec-async feature.

An encoded message consists of two parts:

  • prefix: 4 bytes containing payload length in bytes (big-endian u32)
  • payload: a value of a known type, encoded with bincode
